Redis集群

Created2025-09-22|Updated2025-09-22|技术Redis
|Post Views:

Redis集群 是 去中心化的，对数据进行水平分片，每台机器只会存储部分数据

键空间会被分为16384个槽，每一个key经过CRC16哈希算法，然后堆16384取余，确定他会在哪个槽位。

如果请求的数据正好在这台机器上则返回，如果不在，则该机器会计算出正确的槽位在哪个机器上，然后让请求者自己重新去找那个节点。

Redis主从同步
什么是主从同步： 适用于Reids主从架构（非集群） 为什么要主从架构： Redis在内存里万一掉电了很危险，此外客户端还可以读从节点来分散主节点压力 主从同步方法： 全量同步：slave第一次连接，或很长时间没有连接后，进行全量同步 slave通过PSYNC请求同步，Master异步地生成RDB文件。在此期间所有新来的写操作会被放在复制积压缓冲区。然后再把RDB文件和缓冲区都发给从节点，从节点把这些装进自己的内存 增量同步：适用于后续同步 或 短暂 断联。master会把自己收到的写命令发给所有的slave自己去执行。（异步）==但是这会导致 master和slave的同步不及时（因为为了master为了保证写入的高性能，主从同步是异步的。因此 slave的数据可能是过期的但是这就要看你的应用场景是否能容忍这种不一致：如果你的场景是淘宝商品浏览，或者朋友圈点赞评论，那么即使有客户端拿到了过期数据页无所谓但如果你的场景是银行扣款或者秒杀商城，那就不能接受，可以通过wait命令手动让master节点在主从同步完成后才给客户端响应success
