Redis为什么这么快

Created2025-09-22|Updated2025-09-22|技术Redis
|Post Views:
  1. 基于内存：主存速度100ns，ssd 50us， 机械硬盘 1ms
  2. 事务单线程（不存在竞争与上锁）+[[IO多路复用]]：
    虽然是单线程，但是由于采用了IO多路复用，相当于这个线程不用去处理IO请求，转而由内核的“时间分发器”处理，而单线程可以更安心的处理事务
  3. Redis6.0之后，由于网络请求IO可能太多了，对于处理连接采用了多线程

Redis集群
Redis集群 是 去中心化的，对数据进行水平分片，每台机器只会存储部分数据 键空间会被分为16384个槽，每一个key经过CRC16哈希算法，然后堆16384取余，确定他会在哪个槽位。 如果请求的数据正好在这台机器上则返回，如果不在，则该机器会计算出正确的槽位在哪个机器上，然后让请求者自己重新去找那个节点。
