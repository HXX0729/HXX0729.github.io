Redis集群
Created2025-09-22|技术Redis
Redis集群 是 去中心化的，对数据进行水平分片，每台机器只会存储部分数据 键空间会被分为16384个槽，每一个key经过CRC16哈希算法，然后堆16384取余，确定他会在哪个槽位。 如果请求的数据正好在这台机器上则返回，如果不在，则该机器会计算出正确的槽位在哪个机器上，然后让请求者自己重新去找那个节点。

Redis的应用场景
Created2025-09-22|技术Redis
缓存：减轻数据库负载，提高读写速度 消息队列：通过利用Redis的List和Pub/Sub功能，可以实现轻量级的消息队列 。 分布式锁： 核心实现原理：利用SETNX命令 实现Redis分布式锁的核心，是利用SET命令的一个特殊选项：NX。 SET key value NX：NX是“Not eXists”的缩写。这条命令的含义是：只有当key不存在时，才会设置key的值为value，并返回成功；如果key已经存在，则什么也不做，并返回失败。 这个操作是原子性的，Redis的单线程模型保证了在同一时刻，即使有成千上万个客户端同时发送SETNX命令，也只有一个客户端能成功设置这个key。这就完美地模拟了“抢锁”这个行为。
